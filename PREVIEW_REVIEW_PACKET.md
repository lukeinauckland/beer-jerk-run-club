# Preview Review Packet

Use this when Vercel gives you a preview URL.

## Timebox

Do this in one Pomodoro.

Goal: catch launch-blocking visual and functional issues, not redesign the whole site.

## Open These Pages

Replace `PREVIEW_URL` with the Vercel preview URL:

```text
PREVIEW_URL/
PREVIEW_URL/new-runners
PREVIEW_URL/schedule
PREVIEW_URL/routes
PREVIEW_URL/afters
PREVIEW_URL/404
PREVIEW_URL/robots.txt
PREVIEW_URL/sitemap.xml
PREVIEW_URL/llms.txt
PREVIEW_URL/facts.json
```

## Mobile Checks

Use a real phone if possible.

Look for:

- Can I understand the next run within 10 seconds?
- Does the quick-action strip work?
- Is the hero text readable?
- Does anything overlap?
- Is there horizontal scrolling?
- Are the schedule rows readable?
- Are route maps legible enough?
- Does the photo wall have empty black gaps?
- Are links easy to tap?

## Desktop Checks

Look for:

- Hero crop.
- Typography scale.
- Wall rhythm.
- Route card balance.
- Footer spacing.
- 404 page not looking broken.

## What To Send Back

If something is wrong, send:

1. Page URL.
2. Screenshot.
3. What feels wrong.
4. Device or viewport if you know it.

Example:

```text
/routes on iPhone. Domain stamp overlaps route copy. Screenshot attached.
```

## Launch Blockers

Do not launch if:

- Next run is wrong.
- Location/time is unclear.
- Mobile layout is broken.
- Links are broken.
- Page has horizontal scrolling.
- Text overlaps photos or other text.
- Stale Half Marathon ticket link appears.
- Meetup appears as a public link.
